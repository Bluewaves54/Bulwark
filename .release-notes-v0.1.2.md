## What's New in v0.1.2

### Universal Explicit Logging Levels

All three proxies (PyPI, npm, Maven) now support configurable `log_level` (debug, info, warn, error) and `log_format` (text, json) in their YAML config files. The `BULWARK_LOG_LEVEL` environment variable can override the config file level at startup — ideal for containers and CI pipelines.

### Global Uninstall (`-uninstall-all`)

A single `-uninstall-all` flag removes **all** installed Bulwark proxies (pypi-bulwark, npm-bulwark, maven-bulwark), their configuration files, autostart entries, and PATH entries in one operation.

### Self-Update (`-update`)

The new `-update` flag checks for the latest stable release on GitHub, downloads the correct binary for your platform, and replaces the running executable in-place.

### Automatic PATH Management

The installer now adds `~/.bulwark/bin` to your system PATH during setup (shell profile on Unix, user environment on Windows) and removes it cleanly on uninstall.
